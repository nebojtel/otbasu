# Security check

Проверено перед упаковкой:

- В архиве нет `node_modules`, `.env`, локальных логов и локальной базы JSON.
- Удалены dev-страницы `/__updater` и `/__status`.
- Удалён локальный Node-сервер для патчей.
- Удалены Windows/macOS запускатели для локальной dev-сборки.
- `SUPABASE_SERVICE_ROLE_KEY` не используется в браузерном коде.
- Создание пользователей вынесено в Vercel serverless API.
- Включены RLS-политики Supabase.
- Витрина может читать только опубликованные товары.
- Админка доступна только пользователям с ролями `admin` или `content_manager`.
- Управление пользователями доступно только роли `admin`.
- Загрузка фото ограничена MIME-типами и размером 5 MB на файл.
- Пользовательские тексты выводятся через HTML escaping.
- Ссылки нормализуются; `javascript:` ссылки не используются как рабочие href.
- Добавлены HTTP security headers в `vercel.json`.

Важное ограничение:

Полностью гарантировать, что сайт «невозможно взломать», нельзя. Безопасность зависит от правильной настройки Supabase, секретов Vercel, RLS-политик, домена и доступа к GitHub/Vercel/Supabase. Эта сборка убирает dev-инструменты и переводит хранение/авторизацию на Supabase с RLS.
